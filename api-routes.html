OpenScenes
Internal API Reference

Users

MethodPathDescriptionBodyResponse
GET /api/users List all users — [{id, name, path, reg_date}]
POST /api/users Create a new user {username} {id, name, path, reg_date} — 201
PUT /api/users/:user_id Rename a user {username?} {id, name, path, reg_date}
DELETE /api/users/:user_id Delete a user (wipes their directory: chats + personas) — {ok: true}
GET /api/users/:user_id/picture Serve the user's profile picture (404 if none) — image bytes
PUT /api/users/:user_id/picture Upload/replace the user's picture (multipart, field file) multipart file {ok: true, picture}
DELETE /api/users/:user_id/picture Remove the user's picture — {ok: true}

Personas

MethodPathDescriptionBodyResponse
GET /api/users/:user_id/personas List a user's personas (metadata only) — [{id, name, description, path}]
GET /api/users/:user_id/personas/:persona_id Get a persona with its markdown content — {id, name, description, path, content}
POST /api/users/:user_id/personas Create a persona (writes a .md file in the user's personas dir) {name, description?, content?} {id, name, description, path} — 201
PUT /api/users/:user_id/personas/:persona_id Edit a persona's metadata and/or markdown content {name?, description?, content?} {id, name, description, path}
DELETE /api/users/:user_id/personas/:persona_id Delete a persona (removes the .md file) — {ok: true}

Characters

MethodPathDescriptionBodyResponse
GET /api/characters List all characters — [{id, name, description, path, last_update}]
GET /api/characters/:character_id Get one character with its file contents (persona, context, opening) — {id, name, description, path, last_update, persona, context, opening}
POST /api/characters Create a new character {name, description?, persona?, context?, opening?} {id, name, description, path, last_update} — 201
PUT /api/characters/:character_id Edit a character's metadata and files {name?, description?, persona?, context?, opening?} {id, name, description, path, last_update}
DELETE /api/characters/:character_id Delete a character, its files, and every chat referencing it across all users — {ok: true}
GET /api/characters/:character_id/picture Serve the character's picture (404 if none) — image bytes
PUT /api/characters/:character_id/picture Upload/replace the character's picture (multipart, field file). Bumps last_update. multipart file {ok: true, picture, last_update}
DELETE /api/characters/:character_id/picture Remove the character's picture — {ok: true, last_update}

Models

MethodPathDescriptionBodyResponse
GET /api/models List all models — [{id, name, description, llm_id, path}]
GET /api/models/:model_id Get one model with its rules content — {id, name, description, llm_id, path, rules}
POST /api/models Create a new model config (the llm_id must refer to an existing LLM entry) {name, llm_id, description?, rules?} {id, name, description, llm_id, path} — 201
PUT /api/models/:model_id Edit a model's settings and rules {name?, description?, llm_id?, rules?} {id, name, description, llm_id, path}
POST /api/models/:model_id/duplicate Clone a model (metadata + rules) under "<name> (copy)" — {id, name, description, llm_id, path} — 201
DELETE /api/models/:model_id Delete a model and its rules file — {ok: true}

LLMs

MethodPathDescriptionBodyResponse
GET /api/llms List every registered LLM. Provider is local (GGUF file) or openrouter (HTTP). API keys are never returned in full — only a …last4 preview plus a has_api_key flag. — [{id, name, provider, filename?, size?, model?, has_api_key?, api_key_preview?}]
GET /api/llms/:llm_id Get one LLM entry (key still masked) — same shape as a list item
POST /api/llms Create an LLM. Local: multipart with fields name, provider=local, file (.gguf). OpenRouter: JSON. multipart or {name, provider:"openrouter", model, api_key} public entry — 201
PUT /api/llms/:llm_id Edit name and (for openrouter) model and api_key. An empty/missing api_key keeps the stored one. {name?, model?, api_key?} public entry
DELETE /api/llms/:llm_id Delete an LLM entry. Local entries also delete the underlying .gguf file. Refuses with 400 when any model preset still references the entry. — {ok: true}

Chats

MethodPathDescriptionBodyResponse
GET /api/chats/:user_id List all chats for a user — [{id, character_id, model_id, last_update, history}]
POST /api/chats/:user_id/start Open a chat for a character. If one already exists, returns it; otherwise creates one seeded with the character's opening message. {character_id} {id, character_id, model_id, last_update, history} — 200 (existing) or 201 (created)
GET /api/chats/:user_id/:chat_id Get a chat (metadata + paginated history). Query: ?before=<id>&limit=<n> — returns the latest n messages with id < before. Side effect: bumps last_update. — {id, last_update, history, has_more, total, ...}
PUT /api/chats/:user_id/:chat_id Overwrite the full message history of a chat [{role, content}] {ok: true}
DELETE /api/chats/:user_id/:chat_id Delete a chat entirely (drops the entry from this user's chats.json) — {ok: true}
PUT /api/chats/:user_id/:chat_id/model Set the model associated with a chat {model_id} {ok: true, model_id}
PUT /api/chats/:user_id/:chat_id/messages/:message_id Edit a message's content and/or its selected_index {content?, selected_index?} {id, role, content, selected_index?}
DELETE /api/chats/:user_id/:chat_id/messages/:message_id Delete a single message by id — {ok: true}
POST /api/chats/:user_id/:chat_id/rewind/:message_id Rewind the chat — keep through this message, drop everything after. Query: ?limit=<n> to receive a paginated tail. — {ok: true, history, has_more, total}

Send

MethodPathDescriptionBodyResponse
POST /api/chat/send Send a message and get a reply from the LLM {user_id, chat_id, character_id, persona_id, model_id, message?, new_attempt?, continue_mode?} {reply: string}

Errors — all routes return {error: string} with status 400, 404, or 500.

continue_mode: true — message can be omitted; the LLM continues without a user turn.

new_attempt: true — discards the last assistant message and regenerates.
